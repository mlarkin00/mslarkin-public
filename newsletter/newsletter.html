// GKE Agentic Operations Program

The GKE Agentic Operations Program is a forum for teams engaged in agentic work. The primary goal is to foster collaboration, knowledge sharing, across teams working on agents related to GKE and Kubernetes.

The forum will focus on alignment, strategy, reviews, and demos. 

  • Alignment: There are several GKE orgs working on agentic operations, bringing everyone together to share roadmaps and plans is crucial to keep the organization aligned. The Agentic Operations team will also work to ensure alignment with FSP and GCA.

  • Strategy: Share and discuss industry trends, product and technology strategy.

  • Reviews: Concept, UX, Design, Eng and Product reviews for awareness, collaboration, risk, and alignment.

  • Demos: Opportunity to share and highlight progress.

This will directly support the Transformative GKE Experience to ensure GKE remains aligned to the Northstar goal of becoming agentic.

If you would like to be involved, please reach out to mslarkin@google.com and adamparco@google.com

Design:go/gke-onemcp

Enterprises are investigating the adoption of Agentic AI systems. The Model Context Protocol (MCP) is the emerging industry standard for these agents to discover and interact with tools, prompts, and resources. MCP servers expose external system capabilities (e.g., APIs) in a format AI agents can consume.

Integrating core services like Google Kubernetes Engine (GKE) with MCP is a strategic goal for delivering agentic experiences across Google Cloud.

The GKE MCP surface will include both an OSS MCP server, and the OneMCP server. This hybrid approach uses the OSS MCP server for early testing/design and previews of new MCP features, and the OneMCP server as the GA surface

// GKE Expert Agents (Concept)

go/gkeea-concept

GKE Expert Agents are specialized "Expert" AI agents, operating autonomously and/or with human-in-the-loop (HITL) guardrails, supporting the day-to-day operations of customers' GKE environments. These expert agents are scoped to particular domains and leverage real-time data and opinionated best practices to execute intelligent actions.

The GKE Ops Agent will act as the GKE Expert subagent for Gemini Cloud Assist agents (Investigations Agent, Design Agent). These agents will send GKE-related tasks/requests to the GKE Ops Agent, which will handle the planning/routing to Expert Agents.

GKE Ops Agent: Provides orchestration for requests to Expert Agents.  It will handle task planning and tracking, routing/assigning queries to the appropriate Expert Agent, and synthesizing responses from each Expert Agent (as necessary) to provide a response to the client agent.

GKE Expert Agents: Domain-scoped agents that fulfill requests related to their specific domain.

OneMCP:  Expert Agents leverage MCP tools/prompts/resources as necessary to interact with GCP services.

Agents

Upgrades

The GKE Upgrade agent tackles one of the highest-risk operations in cluster management: version upgrades. It acts as an intelligent co-pilot, designed to prevent downtime caused by API deprecations, webhook incompatibilities, or insufficient node readiness.

It achieves this by performing exhaustive, automated pre-flight checks against the cluster's specific workloads and configuration before the upgrade begins. It:

  • Scans Workloads: Queries the cluster to find any running workloads that are still using Kubernetes APIs scheduled for removal in the target version.

  • Reviews Kubernetes and GKE release notes/changelogs: Checks for deprecations and/or breaking changes that impact the target cluster(s)

  • Tests Admission Webhooks: Proactively provisions a test node on the new version to verify that all custom and third-party admission webhooks (like security policy enforcers) function correctly.

  • Validates Node Pool Readiness: Checks that node pools have sufficient capacity and correct configurations to support a rolling upgrade strategy.

Feature Discovery

The Feature Discovery agent reviews/monitors the customer's workloads and configuration, and recommends leveraging new features, where appropriate.
